# PaperTrade

A risk-free **paper-trading** web app for learning to invest in the US stock
market. Sign in with Google, get a virtual **$10,000**, and buy and sell real
stocks at real market prices — without risking a cent. Track your portfolio,
review your trade history, explore top market movers, and (optionally) get an
AI-generated debrief after every sale.

> Installable as a mobile app (PWA) and deployable to Vercel.

---

## Features

- 🔐 **Google sign-in** with per-user data isolation — every user gets their own private portfolio.
- 💵 **$10,000 virtual starting balance** per account.
- 📈 **Real market data** — live-ish quotes, 30-day price charts, and company news.
- 🛒 **Buy & sell** with real prices, including:
  - **Adding to a position** (averages your cost basis automatically).
  - **Partial sells** (sell some shares or all).
- 📊 **Dashboard** — portfolio value, cash, invested amount, unrealized P&L, and a **portfolio-value-over-time chart**.
- 🔎 **Explore** — search any ticker, plus a computed **Top Movers** list (gainers/losers) with a toggle.
- 🧾 **Trade history** — full log of every buy/sell with realized P&L.
- 🤖 **AI trade debrief** (optional) — a short post-trade coaching note generated by Claude after each sell.
- 🔁 **Reset** — wipe your portfolio back to a fresh $10,000 anytime.
- 📱 **Mobile-first PWA** — responsive layout, app-style bottom tab bar, installable to your home screen.

## How it works

- **Auth & security:** Google OAuth through Supabase. A Next.js proxy (middleware)
  refreshes the session and redirects unauthenticated users to `/login`. Every
  database table carries a `user_id`, and Postgres **row-level security** ensures
  users can only ever read/write their own rows.
- **Data flow:** the browser calls same-origin `/api/*` routes; those routes use a
  cookie-bound Supabase server client, so each request is scoped to the signed-in
  user. The trade engine (`lib/trade-engine.ts`) handles buy/sell logic, cost-basis
  averaging, partial sells, and portfolio snapshots.
- **Rate-limit friendly:** the Top Movers list is derived from Polygon's
  **grouped-daily** endpoint (every US stock in one request, cached) rather than
  many per-ticker calls, to stay within the free tier.

### Project structure

```
app/
  page.tsx                 Dashboard (portfolio, chart, positions, reset)
  explore/                 Search + Top Movers + stock detail
  trade/[symbol]/          Buy / sell a stock

  trade-engine.ts          Buy/sell logic
  polygon.ts · finnhub.ts  Market data wrappers
  claude.ts                AI debrief
proxy.ts                   Session refresh + route protection
database*.sql              Schema migrations (run in order)
```

## Getting started

### 1. Prerequisites

- Node.js 18+ and npm
- A free [Supabase](https://supabase.com) project
- API keys: [Polygon.io](https://polygon.io) and [Finnhub](https://finnhub.io) (both have free tiers)
- A Google Cloud project for OAuth (free)
- *(optional)* an [Anthropic API key](https://console.anthropic.com) for AI debriefs

### 2. Install

```bash
git clone https://github.com/Dark10701/Trading-stocks.git
cd Trading-stocks
npm install
```

### 3. Configure environment

Copy the example file and fill in your values:

```bash
cp .env.example .env.local
```

| Variable | Required | Purpose |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Supabase project URL (base, no `/rest/v1/`) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | Supabase anon/public key |
| `POLYGON_API_KEY` | ✅ | Stock prices, charts, search, movers |
| `FINNHUB_API_KEY` | ✅ | Company news |
| `ANTHROPIC_API_KEY` | ➖ | Optional — enables AI trade debriefs |
| `NEXT_PUBLIC_APP_URL` | ➖ | Only for custom domains in production |

### 4. Set up the database

In the Supabase SQL editor, run these files **in order**:

1. `database.sql` — core tables (portfolio, positions, trades)
2. `database-rls-fix.sql` — initial RLS + starting balance
3. `database-v2.sql` — portfolio value snapshots (chart)
4. `database-v3.sql` — per-user ownership + auth RLS

### 5. Enable Google sign-in

Follow **[AUTH-SETUP.md](AUTH-SETUP.md)** — create Google OAuth credentials and
enable the Google provider in Supabase (~10 minutes, one-time).

### 6. Run

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000), sign in with Google, and start trading.

## Notes & limitations

- **Stock data is free-tier**, so prices reflect the most recent completed trading
  day rather than live intraday ticks.
- **Top gainers/losers** are real but, by nature, are often smaller, less-familiar
  companies (mega-caps rarely move double digits in a day). A liquidity filter
  keeps them tradable.
- Paper trading only — **no real money is ever involved.**
