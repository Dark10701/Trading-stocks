# PaperTrade

A risk-free paper-trading web application for learning to invest in the US stock
market. Users sign in with Google, receive a virtual balance of $10,000, and buy
and sell real stocks at real market prices without risking any money. The
application tracks each user's portfolio, records a full trade history, surfaces
top market movers, and can optionally generate an AI-written debrief after each
sale.

The app is a mobile-first Progressive Web App (installable to a phone's home
screen) and is ready to deploy to Vercel.

---

## Features

- **Google sign-in with per-user data isolation.** Every user has their own
  private portfolio, enforced at the database level.
- **$10,000 virtual starting balance** for each account.
- **Real market data** — quotes, 30-day price charts, and company news.
- **Buy and sell** at real prices, including adding to an existing position
  (cost basis is averaged automatically) and partial sells.
- **Dashboard** showing total value, cash, invested amount, unrealized P&L, and a
  portfolio-value-over-time chart.
- **Explore** page with ticker/company search and a computed Top Movers list
  (gainers and losers, with a toggle).
- **Trade history** — a complete log of every transaction with realized P&L.
- **AI trade debrief (optional)** — a short post-trade coaching note generated by
  Claude after each sale.
- **Reset** — restore a portfolio to a fresh $10,000 at any time.
- **Mobile-first PWA** — responsive layout, app-style bottom navigation, and
  home-screen installation.

## Architecture

- **Authentication and security.** Google OAuth is handled through Supabase. A
  Next.js proxy (middleware) refreshes the session on each request and redirects
  unauthenticated users to `/login`. Every database table carries a `user_id`, and
  Postgres row-level security guarantees that users can only read and write their
  own rows.
- **Data flow.** The browser calls same-origin `/api/*` routes, which use a
  cookie-bound Supabase server client so each request is scoped to the signed-in
  user. The trade engine (`lib/trade-engine.ts`) handles buy/sell logic, cost-basis
  averaging, partial sells, and portfolio value snapshots.
- **Rate-limit efficiency.** The Top Movers list is derived from Polygon's
  grouped-daily endpoint, which returns every US stock in a single cached request,
  rather than many per-ticker calls. This keeps the app within free-tier limits.

### Project structure

```
app/
  page.tsx                 Dashboard (portfolio, chart, positions, reset)
  explore/                 Search, Top Movers, and stock detail
  trade/[symbol]/          Buy / sell a stock

  login/                   Google sign-in
  auth/callback/           OAuth code exchange
  api/
    portfolio/             Portfolio and value history
    trade/                 Execute buy/sell
    history/               Trade log
    reset/                 Reset portfolio
    debrief/               AI trade debrief
    stocks/                quote, history, search, news, top (movers)

  trade-engine.ts          Buy/sell logic
  polygon.ts, finnhub.ts   Market data wrappers
  claude.ts                AI debrief
proxy.ts                   Session refresh and route protection
database*.sql              Schema migrations (run in order)
```

## Getting started

### Prerequisites

- Node.js 18 or later, and npm
- A free Supabase project
- API keys for Polygon.io and Finnhub (both offer free tiers)
- A Google Cloud project for OAuth
- Optionally, an Anthropic API key for AI debriefs

### Installation

```bash
git clone https://github.com/Dark10701/Trading-stocks.git
cd Trading-stocks
npm install
```

### Configure environment

Copy the example file and fill in your values:

```bash
cp .env.example .env.local
```

| Variable | Required | Purpose |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL (base, no `/rest/v1/`) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Supabase anon/public key |
| `POLYGON_API_KEY` | Yes | Stock prices, charts, search, movers |
| `FINNHUB_API_KEY` | Yes | Company news |
| `ANTHROPIC_API_KEY` | No | Enables AI trade debriefs |
| `NEXT_PUBLIC_APP_URL` | No | Only needed for custom domains in production |

### Set up the database

In the Supabase SQL editor, run these files in order:

1. `database.sql` — core tables (portfolio, positions, trades)
2. `database-rls-fix.sql` — initial row-level security and starting balance
3. `database-v2.sql` — portfolio value snapshots (chart)
4. `database-v3.sql` — per-user ownership and authentication policies

### Enable Google sign-in

Follow [AUTH-SETUP.md](AUTH-SETUP.md) to create Google OAuth credentials and enable
the Google provider in Supabase. This is a one-time setup that takes roughly ten
minutes.

### Run

```bash
npm run dev
```

Open http://localhost:3000, sign in with Google, and start trading.

## Notes and limitations

- Stock data uses free-tier endpoints, so prices reflect the most recent completed
  trading day rather than live intraday ticks.
- The top gainers and losers are computed from real data, but by nature they are
  often smaller, less-familiar companies, since large-cap stocks rarely move by
  double digits in a single day. A liquidity filter keeps the results tradable.
- This is paper trading only. No real money is ever involved.
